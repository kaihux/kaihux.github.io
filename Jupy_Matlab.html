Though MATLAB is proprietary, it is still very popular in academia research and among college students who picked up research projects or simply need to finish an assignment. Matlab is a language relatively easy to pick up and less grammatically strict compared with other programming languages, like C, C++ and Python. Since the advent of Matlab, many useful and specialized toolboxes and functions have been written for matlab, which makes it very appealing to researchers who need easy-to-use tools specifically tailored to solve their problems (the first-order priority for researchers in science and engineering is not code writing, but to attack the central scientific/engineering questions with tools they can assemble). For myself, the big reason why I hang onto Matlab is mostly because of TopoToolbox, a very handy matlab-based toolbox for topographic analysis.

Mathworks packs the Matlab language and its “heavy” GUI into a gigantic software, the GUI makes it convenient to manage files and variable workspace for users who are not very comfortable with command-lines. However, the “heavy coat” of GUI makes MATLAB more like a software package than a programming language . Jupyter notebook is an interactive development interface to a wide variety of programming languages. The architecture and philosophy of Jupyter Notebook is, in my opinion, to let people focus on quality code writing and commenting, giving people a fresh feeling with its clean interface.

Running Matlab in Jupyter Notebook will strip Matlab down to its core of a programming language.

OK, here is the end of the digression and comes the essence of this writing.

Running Matlab in Jupyter Notebook is made possible by Matlab Engine API for Python. The tutorial is based on https://am111.readthedocs.io/en/latest/jmatlab_install.html

First of all, make sure you have a licensed version of matlab working on your machine and I can only speak to my experience with Matlab 2018b on MacOS.

1. Install Jupyter Notebook.

https://jupyter.org/install

2. Install Matlab kernel for Jupyter.

Python side configuration.

Different versions of Matlab interfaces with different python versions (link). The default python version on your system is not necessarily compatible with your matlab api for python.

My MacOS has a python version of 3.7, but the matlab build I have on my mac only supports up to 3.6. Since I don’t want to go to the length of downgrading my system default python version, it is necessary to create a virtual environment of python 3.6.

conda create -vv --prefix /Users/kai/Documents/MATLAB/Jupyter_matlab/jmatlab python=3.6 jupyter

 

The path after --prefix is where you choose to install the environment.

Now we activate the environment and stay in this environment for the rest of this tutorial.

conda activate /Users/kai/Documents/MATLAB/Jupyter_matlab/jmatlab

 

Then, install the Matlab kernel for Jupyter. （Archived as of 04/27/2020）

pip install git+https://github.com/Calysto/matlab_kernel #this is a development version
python -m matlab_kernel install

pip install matlab_kernel might encounter an error of not finding a version that meets the requirement.

Check if there is a matlab version available in Jupyter Notebook

jupyter kernelspec list

You should be able to see matlab listed.

3. Matlab-side configuration

Find the directory to which the matlab is installed on MacOS, it is usually at “/Applications/MATLAB_2018b.app”

Go to the python engine subdirectory and install the python engine

cd /Applications/MATLAB_2018b.app/extern/engines/python
python setup.py install

Executing the second line requires administrator permission. If you are doing this on a server of which you are not an administrator, this will be problematic.

So you will have to do it slightly differently.

python setup.py build --build-base=$HOME/tmp/build install

This will build in your personal home directory, where you have read and write privileges.

By the way, it is not easy sometimes to find the path to the installation directory of Matlab on a linux server, here is how

open matlab and in its command line type:

matlabroot

The root directory of matlab will be returned.

From the virtual environment you set, you are all set to run matlab in Jupyter Notebook

jupyter notebook from the terminal.
